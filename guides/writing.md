# 글쓰기

톤·구조·출처 규칙.

이 문서는 **명세**다. 기존 발행글 6편에서 역추출했다 (Network 2, ComfyUI 2, 그 외).
`lint_post`와 팩트체커가 여기 정의된 규칙 중 검증 가능한 것을 강제한다.

---

## 1. 톤 — 세 가지

글의 성격에 따라 톤을 고른다. **Kind 태그와 대응한다.**

| Kind | 톤 | 서술 |
|---|---|---|
| (없음) · `정리` | **정의 직술** | 현재형 단정. 1인칭 없음 |
| `입문` | **절차 안내** | 현재형. 1인칭 허용 |
| `트러블슈팅` · `회고` | **1인칭 경험** | 과거형. 1인칭 중심 |

**한 글에 한 톤.** 섞지 않는다.

### 1.1 정의 직술

**주어가 개념이다.** 세상의 사실을 진술한다.

```
사설망(LAN)은 물리적으로 통제 가능한 범위 안에서만 동작한다.
IPSec은 네트워크 계층(L3)에 보안 서비스를 제공하며, 패킷 단위로 적용된다.
AH는 무결성을 담당하고, ESP는 기밀성을 담당한다.
```

- 현재형 단정: `~한다`, `~이다`
- 순서: 개념 → 정의 → 메커니즘 → 활용

#### 1인칭 — 두 가지를 구분한다

**화자로서의 "나"는 쓰지 않는다.** 사실을 의견으로 격하시킨다.

```
✗ 내가 보기에 IPSec은 L3에서 동작한다.
✓ IPSec은 L3에서 동작한다.
```

**설명 속 등장인물로서의 "나"는 오히려 좋다.** 독자를 예시 안으로 끌어들인다.

```
✓ 내 PC의 IP가 192.168.0.10인데도 인터넷이 잘 되는 이유는...
✓ 내가 먼저 말을 걸어야 표에 기록이 남고, 그 기록이 있어야 응답이 돌아온다.
✓ DPI가 무제한 허용되면 내가 주문한 물건까지 전부 들여다보게 된다.
```

앞의 것은 **판단의 주체**이고, 뒤의 것은 **예시의 참여자**다. 뒤의 것은
추상적 개념을 독자의 경험으로 끌어내리는 장치다.

**해당**: Network, DSA, OS, C 강의 트랙

### 1.2 절차 안내

**선택과 안내다.** 화자가 필요하다 — 누가 그렇게 판단했는지가 드러나야 한다.

```
이 글은 포터블 빌드로 진행한다.
내가 자주 쓰는 해상도는 아래 정도다.
부정 프롬프트는 위처럼 고정해두고 긍정 프롬프트만 바꿔가며 쓰는 방식이 편하다.
```

- 현재형이되 **1인칭 허용**: `내가 자주 쓰는`, `이 글은`
- **선택의 근거를 밝힌다.** 대안을 표로 비교하고 고른 이유를 쓴다
- 독자를 데리고 간다: `~하면 된다`, `~해본다`

**해당**: 설치·설정 가이드, 도구 사용기

> 입문 글에서 1인칭을 빼면 딱딱해진다. *"포터블 빌드가 가장 적합하다"*로 바꾸면
> **누가 그렇게 판단했는지가 사라진다.**

### 1.3 1인칭 경험

**과거형. 내가 겪은 일이다.**

```
설치 중 Next 버튼이 비활성화됐다.
EFI 파티션을 만들었는데도 넘어가지 않았다.
```

- 과거형: `~했다`, `~였다`
- 문제 → 시도 → 실패 → 원인 → 해결 순
- **막힌 지점을 감추지 않는다.** 그게 이 톤의 가치다

**해당**: 트러블슈팅, 회고

## 2. 출처

### 2.1 원칙

**검증 가능한 주장에는 출처를 단다.** 글의 종류와 무관하다.

| 대상 | 출처 필요 |
|---|---|
| RFC 번호, 표준 스펙 | ✅ |
| 성능 수치, 파라미터 수, 용량 | ✅ |
| 버전별 차이, 지원 범위 | ✅ |
| 역사적 사실, 날짜 | ✅ |
| 라이선스 조건 | ✅ |
| 1인칭 경험·환경 관찰 | ❌ 불필요 |
| 개념 설명, 정의 | ❌ 불필요 |

```
✗ Anima는 20억 파라미터로 비교적 가볍다.
✓ Anima는 20억 파라미터로 비교적 가볍다. ([모델 카드](https://huggingface.co/...))

✗ Anima는 512×512부터 1536×1536까지 지원한다.
✓ Anima는 512×512부터 1536×1536까지 지원한다. ([공식 문서](...))
```

> **기준은 "강의 기반이냐"가 아니라 "검증 가능하냐"다.** 강의도 틀릴 수 있다.

### 2.2 강의 내용

강의에서 배운 **개념·설명**은 Source 태그로 충분하다. 본문에 링크를 달지 않는다.

**다만 강의가 말한 검증 가능한 사실은 다르다.** RFC 번호, 스펙, 수치는
강의에서 들었더라도 **직접 확인하고 출처를 단다.**

### 2.3 출처를 댈 수 없으면

**단정하지 않는다.** 둘 중 하나를 택한다.

```
완화:  "~로 알려져 있다", "일반적으로", "내 환경에서는"
삭제:  그냥 뺀다
```

> **모른다고 말하는 것이 지어내는 것보다 낫다.**
>
> 기술 블로그에 없는 출처를 지어낸 주장이 실리면 그건 오염이다.
> 축적된 글이 많을수록 비용이 커진다.

## 3. 구조

### 3.1 섹션

- **`##`가 최상위 섹션.** `#`는 제목에만 쓴다
- `###`는 `##` 아래 하위 절
- `####` 이하는 쓰지 않는다

### 3.2 도입

두 형태 중 하나.

| 형태 | 언제 |
|---|---|
| `## 들어가며` | 개념 글. 문제 제기 → 이 글이 다룰 범위 |
| 무제 단락 | 가이드·시리즈 후속. 바로 본론 맥락 |

**도입 마지막에 이 글의 범위를 밝힌다.**

```
이 글에서는 VPN이 무엇을 묶고, 어떤 기술로 그것을 구현하며,
어떤 형태로 동작하는지, 그리고 그 힘이 어떻게 악용되는지까지 살펴본다.
```

### 3.3 마무리 — 필수

**모든 글은 `## 마무리` 또는 `## 정리`로 끝난다.**

**요약이 아니라 재정리다.** 본문을 반복하지 말고, **읽는 법**을 한 줄로 압축한다.

```
✓ 네트워크 보안 솔루션은 종류가 많지만, 읽는 법은 두 축으로 좁혀진다.
✗ 이 글에서는 NAC, 방화벽, WAF, IPS, NIDS에 대해 알아보았다.
```

시리즈물이면 마지막 단락에서 **다음 글을 예고한다.**

### 3.4 표

**비교·분류·파라미터는 산문 대신 표로 쓴다.**

- 선택지 비교 (설치 방법 4가지)
- 분류 (프로토콜 3종, 방화벽 3종)
- 파라미터 (KSampler 설정)
- 매핑 (계층 ↔ 데이터 단위 ↔ 식별자)

산문으로 나열하면 읽히지 않는다.

### 3.5 이미지

**먼저 판단한다 — 이 개념을 그림으로 낼지.** 썸네일 오브젝트 선정(`thumbnails.md`
§2.4)과 같은 부류다. 기계가 못 하는 의미 판단이라 Claude가 한다. 강제는 아래 원장이 한다.

그림으로 낼 것:

- **멘탈 모델·2축 매트릭스(2×2)** — 두 축의 교차가 의미면 표보다 배치가 낫다
- **다단계 과정·상태 전이(before/after)** — 순서·인과가 핵심일 때
- **공간적·구조적 관계** — 자료구조 결합, 계층

그림이 아닌 것:

- 비교·분류·파라미터·매핑 → **표** (§3.4)
- 선형 서사·정의 → **산문**
- 정량 정확도 → matplotlib (SVG 아님, `diagram-system.md` §6)

**결정을 원장에 남긴다.** frontmatter 바로 다음(없으면 최상단)에 아래 블록을 두고, 각
핵심 개념을 그림/표/산문 중 무엇으로 낼지 한 줄씩 이유와 함께 적는다. frontmatter
**앞**에 두면 안 된다 — `\A---` 앵커가 깨져 카테고리·태그가 안 읽힌다. `lint_post` POST-15가 **원장 존재를 검사한다**
— 내용 판단이 아니라 존재 검사다 (POST-13이 파일명 존재만 보는 것과 같은 teeth). 판단은
Claude, 강제는 코드. `build`가 발행본에서 이 블록을 떼어낸다 (렌더에 안 보인다).

```markdown
<!-- DIAGRAM-LEDGER
guides/sensors 2×2 → 그림 | 두 축의 교차라 배치가 의미를 전달, 표로는 안 보임
파이프라인 순서 → 산문+코드 | ASCII 흐름도로 충분
최대 부분합 예제 → 그림 | dp 배열을 채우는 순서가 핵심
-->
```

한 줄 형식은 `<개념> → <결정> | <이유>` 를 권장하되, 검사는 형식을 강제하지 않는다
(오탐 방지). 그림이 하나도 없는 글도 "산문으로 충분"을 한 줄 남긴다 — 판단을 건너뛰지
않게 하는 게 원장의 목적이다. 주석 안에 `--`(하이픈 둘)를 쓰지 않는다 (HTML 주석이 깨진다).

placeholder 문법:

```markdown
[IMG: 설명]
```

placeholder로 쓰고, 발행 시 PNG를 드래그앤드롭한다.
설명은 **무엇을 보여주는 그림인지** 구체적으로 쓴다.

자작 다이어그램은 파일명을 함께 적는다. **첫 토큰을 파일명(소문자_밑줄)으로 둔다** —
`lint_post`가 이 파일명을 `diagrams/` 에 대조한다 (POST-13). 오타·드리프트를 잡는다.

```markdown
[IMG: net_vpn_concept — VPN = Virtual + Private 분해. 사설망이 인터넷 위로 확장되는 개념도]
```

애니메이션 GIF 도 같다: `[IMG: dp_longest_path (애니메이션) — ...]`. 첫 토큰 `dp_longest_path`
가 `diagrams/dsa/dp_longest_path.gif` 로 존재하면 통과.

### 3.6 Callout

**Obsidian 문법으로 쓴다.** HTML을 직접 쓰지 않는다 (변환기가 처리한다).

```markdown
> [!important] VPN의 핵심
> GtoE VPN에서 PC의 IP 주소는 **두 개**가 된다.
```

- 본문 흐름에서 **비켜난** 주의·보조는 callout
- **핵심은 본문에.** callout으로 빼면 흐름이 끊긴다
- 상세는 `guides/callouts.md`

## 4. 금지

### 4.1 대시 (—)

**본문 산문에서는 쓰지 않는다.** 구조적 구분자로는 쓴다.

**허용 — 구조적 구분자**

```
✓ # 인터넷 공유기와 NAT — 하나의 주소로 여럿이 인터넷을 쓰는 법     (제목 부제)
✓ ## NAT란 — 주소와 포트를 바꿔치기하는 기술                        (섹션 부제)
✓ [IMG: Symmetric NAT #1 — 출발지 IP 192.168.0.10...]              (라벨 구분)
```

**금지 — 본문 산문**

```
✗ Inline, Out of path, Proxy—이 셋 중 하나에 속한다
✗ Sandbox·MPS·ESM·SIEM까지—결국 이 두 축 안에서 읽힌다
✗ P2P를 어떻게 다시 뚫는지 — 이 세 가지를 차례로 풀어본다

✓ Inline, Out of path, Proxy. 이 셋 중 하나에 속한다
✓ Sandbox·MPS·ESM·SIEM까지 결국 이 두 축 안에서 읽힌다
✓ P2P를 어떻게 다시 뚫는지. 이 세 가지를 차례로 풀어본다
```

산문의 대시는 `humanize-korean`의 **J: 시각 장식 남용**에 해당한다.
문장을 끊거나(`.`), 쉼표로 잇거나, 문장을 나눈다.

**가운뎃점(·)은 허용한다** — 나열에 쓴다 (`NAC·방화벽·WAF`).

### 4.2 AI 티

**`humanize-korean` 스킬이 잡는다** (`im-not-ai` repo — 도구 이름은 humanize-korean).
대표적인 것:

| 카테고리 | 예 |
|---|---|
| A 번역투 | `~를 통해`, `~에 있어서`, `~되어진다` |
| C 구조적 패턴 | 기계적 `첫째/둘째/셋째`, 과도한 불릿·헤딩 |
| D AI 관용구 | `결론적으로`, `시사하는 바가 크다`, `주목할 만하다` |
| G Hedging | `~할 수 있을 것으로 보인다` |
| H 접속사 남발 | 문두 `또한/따라서/즉` 연속 |
| J 시각 장식 | 과도한 **볼드**, 대시 남발 |

**발행 직전에 `humanize-korean`을 돌린다.** 통과한 파일이 최종본이다.

### 4.3 극적 수사 어휘 (POST-12)

**본문은 담백하게.** "영화 대본·드라마 대사" 같은 극적 수사는 쓰지 않는다.

- **수사 의문문**(`왜일까`, `무엇이 갈랐을까`)은 던지지 말고 답을 바로 진술한다
- **극적 은유·단정**(`~의 심장이다`, `열쇠는 ~것이다`, `못 박는다`)은 사실로 내린다
- **긴장 조성 동사**(`지배한다`, `무너진다`)는 평이한 서술어(`결정한다`, `성립하지 않는다`)로
- **동작을 극적으로 연출하는 서술어**(`박아 넣는다`, `딸려 온다`, `뒤집는다`, `쥐고 있다`)는 사실 그대로(`채운다`, `함께 설정된다`, `달라진다`, `쓰고 있다`)로. 단 `뒤집`처럼 정당한 기술 용법(배열을 뒤집는다)이 있는 어휘는 목록에 넣지 않고 사람이 본다

아래는 **자라는 목록**이다 (POST-05a 복수형 허용목록·팔레트 승격과 같은 promote 모델).
`lint_post`가 이 블록을 파싱해 본문 산문에서 어휘를 찾으면 WARN 한다. WARN 이 반복되면
여기 한 줄을 추가한다.

> 막으면 린터를 끄게 된다. 넓은 "톤"은 여전히 사람이 본다 — 여기 있는 특정 관용구만
> 기계가 잡는다.

## 5. 파이프라인에서의 위치

```
drafts/<slug>.md        ← 이 문서의 규칙으로 쓴다
      ↓ make check       기계 검사 (링크·태그·카테고리)
      ↓ make factcheck   출처 없는 주장 적발 (§2)
      ↓ humanize-korean  AI 티 제거 (§4.2)
★ 최종 파일
      ├→ Obsidian 아카이브
      └→ make build → posts/  (callout → HTML)
```

**카테고리·태그는 초안 맨 위 frontmatter 로 선언한다.** `make check`(= `lint-post`)가
여기서 읽어 POST-01~05 를 강제한다. `make build` 가 발행본에서 이 블록을 떼어내므로
Tistory 에 새지 않는다. `--category`/`--tags` 를 주면 frontmatter 를 덮어쓴다.

```yaml
---
category: DSA
tags: [DynamicProgramming, 동적계획법, Memoization, 입문]
---
```
