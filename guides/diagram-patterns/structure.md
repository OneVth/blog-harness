# 구조 다이어그램

두 자료구조의 결합. 한쪽이 다른 쪽을 참조하는 관계.

**언제**: 해시+리스트, B-tree 페이지+엔트리, 패킷 헤더 필드맵, 캡슐화

## 구성

- 상단: 인덱스 배열 (슬롯)
- 한 슬롯에서 수직 화살표
- 하단: 노드 시퀀스 `[key|next] → [key|next] → [key|null]`
- 노드 박스는 가운데 세로선으로 데이터/포인터 분리

## 변형

### 필드맵 (헤더 구조)
32bit × N행 격자. 강조할 필드는 색으로.

### 캡슐화 (중첩)
Frame ⊃ Packet ⊃ Segment. 마트료시카처럼 겹친 박스.
Header/Payload의 **상대성**을 보이는 것이 핵심.

### 색 분해
한 구조의 각 부분을 색으로 나누고, 실제 예시를 **같은 색**으로 대응.
형식 ↔ 예시 매핑에 효과적. (예: URI 구조 → 실제 URL)
