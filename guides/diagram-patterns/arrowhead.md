# 화살촉

**사선·곡선 화살표를 그릴 때 반드시 읽는다.** 눈대중으로 찍으면 각도가 어긋난다.

린터(`lint_svg`)가 이 공식 위반을 검사한다.

---

## 1. 왜 marker를 안 쓰나

SVG `<marker>`의 `orient="auto"`가 신뢰할 수 없다:

| 대상 | 결과 |
|---|---|
| `<line>` 단일 직선 | 정상 |
| `<polyline>` 마지막 segment | **깨짐** |
| `<path>` (Bezier) 끝점 | **깨짐** |

**그래서 화살촉을 `<polygon>`으로 직접 그린다.**

| 화살표 | 방식 |
|---|---|
| 수평·수직 단일 직선 | `<line>` + marker (안정) |
| 점선 직선 | `<line>` + polygon |
| 꺾인 선 | `<polyline>` + polygon |
| 곡선 (호) | `<path>` Q + polygon |
| 양방향 | `<line>` + polygon 2개 |

---

## 2. 수평·수직 화살촉

**아래 방향** (끝점 `(x, y)`):
```svg
<polygon points="x-6,y x+6,y x,y+11" fill="..."/>
```
폭 12px, 높이 11px.

**오른쪽**:
```svg
<polygon points="x,y-5 x,y+5 x+10,y" fill="..."/>
```

**왼쪽**:
```svg
<polygon points="x,y-5 x,y+5 x-10,y" fill="..."/>
```

---

## 3. 사선·곡선 공식

수평·수직이 아닌 **모든** 화살표는 이 공식으로 polygon 3점을 계산한다.

```
1. 팁 좌표      P = (px, py)

2. 방향 단위벡터 d = (dx, dy)
     사선: normalize(끝 - 시작)
     곡선: normalize(끝점 - 제어점)      ← 끝점의 접선

3. 수직벡터     perp = (-dy, dx)

4. 밑변 두 점   P - 12·d ± 5·perp
     = (px - 12dx - 5dy,  py - 12dy + 5dx)
       (px - 12dx + 5dy,  py - 12dy - 5dx)

5. polygon = 팁 P + 밑변 두 점

6. line/path는 팁이 아니라 밑변(P - 12·d)에서 끊는다
```

**6번을 빼먹으면 선이 화살촉을 뚫고 나온다.**

### 예

팁 `(224, 172)`, 시작 `(190, 140)`:

```
d    ≈ (0.728, 0.685)
perp ≈ (-0.685, 0.728)

밑변 = (224 - 8.7 ∓ 3.4,  172 - 8.2 ± 3.6)
     = (211.8, 167.4) 와 (218.7, 159.5)

polygon: "224,172 211.8,167.4 218.7,159.5"
line 끝: (215, 163)      ← 팁이 아니라 밑변
```

## 4. 양 끝 간격

- **출발**: 박스 **바깥**에서 시작한다. 박스 내부에서 시작하면 화살표가 박스를
  뚫고 나온 것처럼 보인다
- **도착**: 박스/셀 **변 앞**에서 멈춘다. 변을 넘으면 안으로 파고든다
- **"닿을 듯 말 듯"이 아니라 명확히 떨어뜨린다**

### 타원 도착

타원에 팁이 파고들지 않게 검산한다:

```
((x - cx) / rx)² + ((y - cy) / ry)² > 1     ← 타원 바깥
```

선 시작점과 화살촉 방향벡터를 **같게** 맞춘다. 안 그러면 선과 촉이 어긋난다.

---

## 5. 자주 하는 실수

| 증상 | 원인 |
|---|---|
| 화살촉이 옆으로 누움 | marker `orient="auto"` 사용. polygon으로 바꿔라 |
| 각도가 미묘하게 어긋남 | 눈대중. §3 공식으로 재계산 |
| 선이 촉을 뚫고 나옴 | line을 팁까지 그음. 밑변에서 끊어라 |
| 촉이 박스 안으로 파고듦 | 팁을 변 위에 놓음. 변 앞에서 멈춰라 |
| fan-out에서 촉이 비뚤어짐 | 밑변 계산 시 `py + 12dy`로 부호 오류. **`py - 12dy`가 맞다** |

---

## 6. 호 (Q 곡선)

옆 슬롯으로 점프하는 작은 호:

```svg
<path d="M x1 y_top Q x_mid (y_top-44) x2 y_top"
      fill="none" stroke="..." stroke-width="1.5" stroke-dasharray="4,3"/>
```

- control point: 시작·끝의 가운데 x, `y_top`에서 위로 44px
- 호 정점: `(x_mid, y_top - 22)`

먼 거리 점프는 곡률을 키운다 (`y_top - 74`).

**곡선 화살촉은 §3의 "끝점 - 제어점"으로 방향벡터를 잡는다.**

> **직각 꺾은선을 우선 검토한다.** 사선·곡선은 화살촉 계산이 까다롭고 라벨이 겹친다.
> "내려갔다 올라온다" 같은 의미는 꺾은선이 더 명확하다.
