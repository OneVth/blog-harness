# 흐름 다이어그램

"X → 함수 → Y → 매핑" 단방향 흐름.

**언제**: DNS lookup, 라우팅 테이블, 컴파일 단계, 요청/응답, 해시 함수

## 구성

- 박스 3~4개 가로 배치 (입력 · 변환 · 중간 결과 · 출력)
- 박스 사이 짧은 가로 화살표
- 도착지에서 결과 영역으로 점선 + polygon 화살촉
- 캡션 한 줄

## 주의

- 화살표는 박스 **바깥**에서 시작, 도착 박스 **변 앞**에서 멈춤 (→ `arrowhead.md`)
- Internet은 타원이 아니라 **겹친 arc path 구름**으로 그린다 (토폴로지 표준)
