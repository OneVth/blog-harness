# 재사용 골격 — User / Kernel / H/W 계층

OS·시스템 동작을 계층으로 설명하는 글의 **공통 배경**.

시스템 콜, 인터럽트, 컨텍스트 스위치, 드라이버, 파일 입출력, 네트워크 계층,
가상화 — 전부 이 골격 위에 얹는다.

**같은 골격을 쓰면 글마다 일관된 시각 언어가 생긴다.**

린터(`lint_svg`)가 divider 좌표와 소켓 접촉을 검사한다.

---

## 1. 구조

```
  User mode
────────────────  divider 1 (들여쓰기 시작)
  Kernel mode
────────────────  divider 2 (왼쪽 끝까지)
  H/W
```

위 두 띠(User + Kernel)가 묶여서 **S/W**다.

---

## 2. 좌표 규격 — 고정

**viewBox 720×480.** 캔버스 상한(900)보다 좁지만 **골격은 720을 유지한다** —
재사용 템플릿이므로 좌표가 고정돼야 한다. 900 본문에서 좁게 렌더되는 것은 문제가
아니다 (가운데 정렬 + 여백).

| 요소 | 좌표 |
|---|---|
| **divider 1** (User/Kernel) | `y=250`, `x 90→690` |
| **divider 2** (S/W ↔ H/W) | `y=350`, `x 30→690` |
| `User mode` 라벨 | `x=96`, `y=62` |
| `Kernel mode` 라벨 | `x=96`, `y=270` |
| `S/W` 라벨 | `x=38`, `y=330` |
| `H/W` 라벨 | `x=38`, `y=370` |

**콘텐츠 영역:**

| 띠 | y 범위 |
|---|---|
| User mode | 78 ~ 244 |
| Kernel mode | 256 ~ 344 |
| H/W | 356 ~ 470 |

가로 사용 폭: `x 90 ~ 690`

> **라벨 들여쓰기가 다른 이유**: `User mode`·`Kernel mode`는 안쪽(x=96),
> `S/W`·`H/W`는 왼쪽 끝(x=38). divider 1은 들여쓴 위치에서 시작하고,
> divider 2는 왼쪽 끝까지 간다. 두 축(모드 / 소프트웨어-하드웨어)이 다른
> 층위임을 시각적으로 드러낸다.

## 4. 소켓 접촉 — 절대 규칙

**divider를 가로지르는 타원/원은 금지다.**

두 영역에 걸치는 요소(소켓 = User/Kernel 진입점)는 **divider에 접하게** 놓는다.
가운데 걸치면 안 된다.

```
cy = divider_y - ry
```

아래변(`cy + ry`)이 divider에 **정확히** 닿아야 한다.

```
      ╭─────╮
      │Socket│
──────╰─────╯──────  ← 아래변이 divider에 접함  ✓ 
      divider

      ╭──┬──╮
──────┼──┼──┼──────  ← 가운데 걸침  ✗ ERROR
      ╰──┴──╯
```

**1px 오차도 안 된다.** `cy=135, ry=16, divider=150`이면 아래변이 151로 **1px 침범**이다.
`cy=134`가 정답이다.

> 소켓이 경계에 "접한다"는 것이 의미다 — User와 Kernel의 **진입점**이지
> 양쪽에 걸친 무언가가 아니다. 걸쳐 놓으면 그 의미가 흐려진다.

타원이 작아 텍스트가 잘리면 `rx`를 키운다 (원 → 타원).

## 5. 변형

### 5.1 2열 비교

같은 골격을 좌우로 놓고 *"같은 띠에서 무엇이 다른가"*를 대비한다.

| 항목 | 값 |
|---|---|
| 좌열 콘텐츠 시작 | **x ≥ 185** — 골격 라벨(x=96)과 간섭 회피 |
| 열 폭 | 240 × 2 (좌 185~425, 우 445~685) |
| 열 헤더 | y=40 부근, 13px bold |

**빈 띠 자체가 메시지가 되도록 한쪽만 채우는 배치를 허용한다.**

### 5.2 VM 블록

가상화 경계를 표현한다. 블로그 본문용은 **Type-2 문법**을 쓴다 (직관 우선).

| 요소 | 스펙 |
|---|---|
| VM 경계 | `stroke="#ff5544" stroke-width="2" stroke-dasharray="4,3"` |
| VM 라벨 | 좌상단 외부, 10px, `#ff5544` |
| 게스트 내부 User/Kernel | 컨테이너 좌우 4px 인셋 라인(sw 1) + 10px `#888` 라벨 |
| 배치 | 게스트 스택은 **호스트 User mode 띠 안**에 |

Type-1을 그릴 땐 전역 divider 없이 **VM 컨테이너마다 내부 축**을 둔다.
정밀한 Type-1 구조는 필요한 글에서만.

### 5.3 커널 표현

- 커널 영역은 **"커널(코어) 박스 + 디바이스 드라이버 박스" 분리**로 그린다.
  중첩이 아니라 **같은 Kernel mode 띠 안의 형제 박스**다
- 단일 OS에서 디바이스 드라이버는 **커널 띠 최하단부**에 놓는다.
  커널 아래 별도 계층이 아니다 (흔한 오해)
- **용어: "디바이스 드라이버"로 통일.** "장치 드라이버" 표기 금지.
  게스트 쪽은 "가상 디바이스 드라이버"

## 6. 배치 원칙

- **블록을 축에 정확히 중앙 정렬한다.** 안 그러면 수직 화살표가 지저분해진다
- **박스 → 박스 연결은 엘보(직각)로.** 사선 지양
- **경로선이 박스 텍스트를 가로지르지 않게** 빈 공간을 찾는다.
  가로 경로는 divider 바로 아래 빈 띠로 빼면 깔끔하다
- 박스 왼쪽이 계층 라벨(`Kernel mode` 등)을 가리지 않게 x 시작을 조정한다
- 추상화 개념(File 등)은 **사각형**으로. 타원 말고. 색은 차분하게 (빨강 지양)
