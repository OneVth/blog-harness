# 시퀀스 다이어그램

한 동작의 단계별 진행. **가장 자주 쓰이는 패턴이다.**

**언제**: 정렬 swap, 힙 sift, 탐색 순서, 완화(relax), 큐/스택 상태 변화

## 구성

- 헤더에 입력 정보 + 컨텍스트
- 진입 화살표로 첫 단계 표시
- 호 화살표(점선) 또는 라벨로 단계 전환
- 동그라미 번호 `①②③` 또는 단계 텍스트
- 도착지·핵심 원소 강조 (`#fff7e6` fill + `#ff5544` stroke 2px)

## 변형

### 압축
2~3단계로 (Initial + 결과), 또는 중간을 `(Step 2 후)` 라벨로 생략.

### 영역 분할
정렬 부분(파랑) / 미정렬 부분(기본)을 색으로 구분.

### 색 추적
객체마다 고유 색을 주고 시간 흐름에서 이동을 추적. (LRU 캐시 등)

### 상태표 시퀀스
좌: 구조 / 우: 단계별 상태표(셀 박스).

**첫 행은 "시작: N 넣음"(초기 상태)부터.** 바로 꺼냄에서 시작하면 어색하다.
큐는 "앞" 마커, 스택은 "top" 마커.

## 주의

- **영역 라벨 y와 슬롯 인덱스 라벨 y가 겹치기 쉽다.** 영역 라벨을 8~10px 위로.
  작성 전에 사전 점검할 것
- 순서 있는 스텝은 **세로로 쌓고** `①②③` 번호. 2열 배치는 읽는 순서가 모호하다
- 표기를 통일한다: 완화는 `"정점: 옛값 → 새값"` 한 형식으로 (`←`·`→`·콜론 혼용 금지)
